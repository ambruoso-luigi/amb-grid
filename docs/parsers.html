Namespace: parsers

parsers

Parser factories used to normalize values before payload/backend submission. Formatters display values, editors let users modify values, validators check whether values are acceptable, and parsers transform values into predictable backend-oriented shapes. Parsers may perform small syntactic checks to avoid incoherent transformations, but they do not replace validators, business rules, permissions, or backend validation.

(static) date(optionsopt) → {Object}

Create a strict date parser. Supports `dd/mm/yyyy`, `dd-mm-yyyy`, `dd.mm.yyyy`, `mm/dd/yyyy`, `mm-dd-yyyy`, `yyyy-mm-dd`, `yyyy/mm/dd`, `yyyymmdd`, `Date`, and the `it`, `iso`, and `legacy` aliases. Separated formats accept one or two digits for day/month; compact `yyyymmdd` stays strictly eight digits. Invalid dates return `null`; empty values follow `allowEmpty`.

(static) decimalToPayload(optionsopt) → {Object}

Normalize a visual decimal value to a payload-oriented decimal string. The parser returns strings instead of JavaScript numbers to avoid precision surprises with decimal and monetary values. It performs syntactic checks only and does not replace decimal validators or backend validation.
